## 第一部分：认识 AI 智能体（Agent）

### 1.1 什么是 AI 智能体？

AI 智能体是一种能够**自主规划和执行多步骤任务**的人工智能程序。与普通的 AI 聊天（比如你问它一个问题，它回答一个答案）不同，智能体可以：

- 📁 **读取和修改文件** —— 它能看懂你项目里的代码和文档
- 💻 **自动执行命令** —— 它可以运行程序、安装依赖、运行测试
- 🔄 **多步骤规划** —— 它会把一个大任务拆分成小步骤，一步一步完成
- 📤 **提交和发布** —— 它能把改好的代码提交到 GitHub，创建 Pull Request（PR）

### 1.2 GitHub Copilot 智能体能做什么？

在 GitHub 上，你可以使用 **GitHub Copilot** 的智能体模式，它能帮你：

| 场景 | 智能体能做的事 |
|------|--------------|
| 写文档 | 根据你的描述，自动生成格式规范的 Markdown 文档 |
| 写代码 | 根据需求描述，生成完整可运行的代码 |
| 修 Bug | 找到代码中的错误并自动修复 |
| 整理项目 | 优化文件结构、添加注释、统一格式 |
| 翻译文档 | 将中文文档翻译成英文，或反之 |
| 发布项目 | 自动创建 PR，协助你完成代码审查和合并 |

## 第二部分：如何提出一个好问题（最关键的一步！）

### 2.1 "好问题"和"坏问题"的区别

很多人第一次使用 AI 时，不知道怎么表达需求，导致结果不满意。其实，**如何提问比 AI 本身更重要**。

❌ **模糊的问题（效果差）：**
> "帮我做一个网站"

✅ **清晰的问题（效果好）：**
> "帮我创建一个个人介绍网页，页面要包含：我的名字（张三）、一段自我介绍（100字左右）、我的技能列表（Python、数据分析、PPT制作）、联系方式（邮箱）。用简洁现代的风格，背景色用浅蓝色。"

### 2.2 好问题的四个要素

```
谁（Who）  + 做什么（What）+ 要求（Requirements）+ 格式/风格（Format）
```

**举例拆解：**

| 要素 | 示例 |
|------|------|
| 做什么 | 写一份产品介绍文档 |
| 内容细节 | 介绍我们的在线学习平台，功能包括：视频课程、在线测验、证书颁发 |
| 目标受众 | 面向完全没有编程经验的大学生 |
| 格式要求 | 用中文，使用标题分级，加上表情符号让内容更活泼 |

**组合后的完整提问：**
> "请用中文写一份面向没有编程经验的大学生的产品介绍文档，介绍我们的在线学习平台。平台功能包括：视频课程、在线测验、证书颁发。文档要使用标题分级，加上表情符号让内容更活泼，篇幅在800字左右。"

### 2.3 常用提问模板

**📄 写文档类：**
```
请帮我写一份【文档类型】，内容关于【主题】，
包含以下部分：【章节1】、【章节2】、【章节3】，
目标读者是【读者群体】，语言风格【正式/轻松/专业】。
```

**🔧 写代码类：**
```
请帮我创建一个【程序/网页/工具】，
功能是【具体功能描述】，
输入是【输入内容】，
输出是【期望结果】，
使用【语言/技术栈（如不确定可省略）】。
```

**✏️ 修改优化类：**
```
请帮我优化以下【文档/代码】，
目标是【更清晰/更简洁/修复错误/添加功能】，
保持【某些内容不变】。
```

## 第三部分：从零开始使用 GitHub

### 3.1 注册 GitHub 账号

1. 打开浏览器，访问 [https://github.com](https://github.com)
2. 点击右上角 **Sign up（注册）**
3. 填写：
   - 邮箱地址（常用邮箱即可）
   - 用户名（英文，这是你的公开名字）
   - 密码
4. 验证邮箱，完成注册

> 💡 **小提示**：用户名会出现在你所有项目的链接里，比如 `github.com/你的用户名/项目名`，建议起一个简洁易记的名字。

### 3.2 创建你的第一个仓库（Repository）

"仓库"就是存放你项目文件的地方，相当于一个文件夹。

1. 登录后，点击页面右上角的 **"+"** 图标
2. 选择 **"New repository"（新建仓库）**
3. 填写信息：
   - **Repository name（仓库名）**：用英文，不要有空格，可以用横线连接，例如 `my-first-project`
   - **Description（描述）**：简短描述这个项目是做什么的（可选）
   - 选择 **Public（公开）** 或 **Private（私密）**
   - ✅ 勾选 **"Add a README file"**（添加说明文件，强烈建议！）
4. 点击 **"Create repository"（创建仓库）**

🎉 恭喜！你已经有了自己的 GitHub 仓库！

## 第四部分：让 AI 智能体帮你完成项目

### 4.1 进入 GitHub Copilot 智能体模式

GitHub 提供了 **Copilot** 功能，可以通过 Issue（问题）来触发智能体。

**步骤：**

1. 进入你的仓库页面
2. 点击上方菜单中的 **"Issues"** 标签
3. 点击右侧绿色按钮 **"New issue"（新建问题）**
4. 在 Issue 标题和内容中，清晰描述你的需求（参考第二部分的提问模板）
5. 提交 Issue 后，在评论区 **@copilot** 或通过界面上的 **"Ask Copilot"** 按钮触发智能体

> ⚠️ **注意**：使用 Copilot 智能体功能需要订阅 GitHub Copilot 服务。个人用户可使用免费试用，学生用户可申请免费版本。

### 4.2 观察智能体工作过程

触发后，智能体会自动：

```
1. 📋 分析你的需求，制定计划
2. 🔍 查看现有文件，了解项目结构
3. ✍️ 创建或修改文件
4. 🧪 运行测试（如有）
5. 📤 提交代码，创建 Pull Request（PR）
```

你可以在 Issue 或 PR 页面实时看到智能体的工作日志，了解它在做什么。

### 4.3 审查智能体的成果（Pull Request）

智能体完成工作后，会创建一个 **Pull Request（PR）**，相当于"申请把改动合并进主项目"。

**如何查看 PR：**

1. 点击仓库页面上方的 **"Pull requests"** 标签
2. 点击最新的那个 PR
3. 切换到 **"Files changed"** 标签，查看智能体做了哪些改动
4. 确认改动符合你的预期

### 4.4 发布你的项目（合并 PR）

如果改动满意，就可以正式发布：

1. 在 PR 页面，滚动到底部
2. 点击绿色按钮 **"Merge pull request"（合并 PR）**
3. 再次点击 **"Confirm merge"（确认合并）**

✅ 完成！你的项目更新已经正式发布到主仓库了！

## 第六部分：常见问题 FAQ

### Q1：我不懂英文，能用中文和智能体沟通吗？

**可以！** GitHub Copilot 支持中文。你可以用中文写需求，它会理解并执行。文件内容也可以是中文。

### Q2：智能体做出来的结果不是我想要的，怎么办？

不用担心！你可以在 PR 的评论区继续说明你的要求，比如：
> "标题改成红色字体" 或者 "第二段内容太长了，请精简到100字以内"

智能体会根据反馈继续修改。

### Q3：我能免费使用 GitHub 和 Copilot 吗？

- **GitHub 仓库**：创建公开仓库完全免费
- **GitHub Copilot**：提供免费试用额度；学生、教师可申请免费使用；个人专业版每月约10美元

### Q4：我的项目别人能看到吗？

- 创建仓库时选择 **Public（公开）**，任何人都可以访问
- 选择 **Private（私密）**，只有你自己（和你邀请的人）可以看到
- 你可以随时在仓库设置中更改这个选项

### Q5：我可以删除仓库吗？

可以。进入仓库的 **Settings（设置）** 页面，滚动到最底部，有 **Delete this repository（删除仓库）** 选项。删除后无法恢复，请谨慎操作。

### Q6：多人可以一起用同一个仓库吗？

可以！这正是 GitHub 最强大的功能之一——**团队协作**。你可以在仓库的 Settings → Collaborators 中邀请其他人加入，大家可以各自修改文件，AI 智能体也能协助每个人完成任务。

## 第七部分：进阶技巧

### 7.1 利用 Issues 管理你的想法

不要只把 Issue 当作"向 AI 提问"的地方。它本质上是一个**任务管理系统**：

- 每个 Issue 代表一个待办任务
- 你可以给 Issue 打标签（Labels），比如"文档"、"新功能"、"Bug"
- 完成后关闭 Issue，保持清晰的项目进展记录

### 7.2 用 GitHub Pages 把项目变成网站

如果你的项目包含网页文件（HTML/Markdown），可以用 **GitHub Pages** 免费发布成网站：

1. 进入仓库 Settings
2. 找到 **"Pages"** 选项
3. 选择分支（Branch）为 `main`，点击 Save
4. 几分钟后，你的网站就会在 `https://你的用户名.github.io/仓库名` 上线！

### 7.3 给你的项目写一个好的 README

README.md 是别人访问你仓库时第一眼看到的内容，相当于项目的"名片"。一个好的 README 应该包含：

- 📌 项目是什么（一句话介绍）
- 🎯 能用来做什么
- 🚀 如何开始使用
- 📞 如何联系作者

> 💡 你可以直接让智能体帮你写：**"请根据这个仓库的内容，帮我写一份完整的 README.md"**
