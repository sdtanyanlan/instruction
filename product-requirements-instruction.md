# 产品需求 Instruction：对话式 AI 图片生成网站

## 文档目标
本 instruction 用于指导需求整理、产品设计和开发范围界定，面向一个基于 Next.js + Node.js 的对话式 AI 图片生成网站。

该网站的核心目标是：
- 用户输入主题后生成图片
- 用户可自行选择使用的模型
- 支持多轮对话
- 支持历史保存
- 自动优化英文 prompt
- 以 Web 聊天页面形式交互

---

## 产品定位
这是一个面向终端用户的对话式 AI 图片生成 Web 应用。

产品形态要求：
- Web 聊天页面
- 聊天式交互而非传统表单
- 聊天输入框附近提供模型选择和图片数量选择
- 支持后续扩展更多图像模型

---

## 技术背景
建议技术栈：
- Next.js
- Node.js
- TypeScript
- Tailwind CSS
- Prisma
- SQLite（MVP）/ PostgreSQL（后续）

---

## 核心功能需求

### 1. 聊天页面
系统必须提供一个聊天页面，包含：
- 左侧历史会话列表
- 中间聊天消息流
- 底部输入区域

### 2. 模型选择
在聊天输入区域附近，必须提供可点击模型选项：
- seedream
- wan2.7

要求：
- 当前选择状态可视化高亮
- 切换后作用于下一次生成

### 3. 图片数量选择
在聊天输入区域附近，必须提供可点击图片数量选项：
- 1
- 2
- 3
- 4

### 4. 图片生成流程
用户输入主题后，系统需完成：
1. 接收用户输入
2. 读取会话上下文
3. 自动将中文输入优化为英文 prompt
4. 根据用户选择的模型调用相应 provider
5. 生成 1~4 张图片
6. 在聊天流中返回结果

### 5. 多轮对话
系统必须支持多轮对话和增量修改，例如：
- 改成黄昏
- 再加一点雾气
- 换成动漫风
- 继续用刚才那个场景

### 6. 历史保存
系统必须保存：
- 历史会话
- 历史消息
- 历史生成记录

### 7. 生成记录保存
每次生成至少保存：
- 原始输入
- 优化后的 prompt
- 模型名称
- 图片数量
- 图片 URL
- 创建时间

---

## 数据模型要求
至少包含以下实体：
- Conversation
- Message
- ImageGeneration
- GeneratedImage

---

## API 范围要求
至少实现：
- POST /api/conversations
- GET /api/conversations
- GET /api/conversations/[id]/messages
- POST /api/chat/generate

---

## MVP 范围
MVP 只要求实现：
- 单聊天页面
- 历史会话列表
- 模型切换
- 图片数量切换
- 多轮对话
- mock 图片生成
- 历史保存
- Prisma + SQLite

当前不要求实现：
- 登录
- 支付
- 图片下载
- 收藏
- 真实模型 API
- 权限系统

---

## UI 要求
- 风格简洁现代
- 适合 AI 聊天产品
- 使用 Tailwind CSS
- 图片结果支持网格展示
- 发送中显示 loading 状态
