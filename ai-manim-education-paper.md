# 人工智能驱动的 Manim 数学动画生成：构建"内容即服务"的教育数字化新范式

## 摘要

教育数字化转型已成为国家战略优先方向，然而当前数学教学中抽象概念可视化资源的制作门槛高、个性化程度低、更新迭代慢等问题仍然突出。本文提出一种"人工智能+Manim"驱动的"内容即服务（Content as a Service, CaaS）"教育数字化新模式，构建了从教师自然语言描述到高质量数学教学动画自动生成的完整技术框架。通过在高等数学课程中的教学实践，验证了该模式在提升学生概念理解深度、增强学习兴趣、提高教师备课效率等方面的显著效果。研究表明，CaaS 模式将教育内容生产从"一次性制作"转变为"按需生成的服务"，为教育数字化转型提供了可推广的新路径。

**关键词：** 人工智能；Manim；内容即服务；教育数字化；数学可视化教学

## 一、引言

### 1.1 研究背景

2022 年以来，教育部实施国家教育数字化战略行动，推动教育资源建设从"有没有"向"好不好"转变。《教育部 2024 年工作要点》明确提出"深入实施国家教育数字化战略行动，以智能化赋能教育治理和教学改革"。在此背景下，如何利用新兴技术提升教学质量、促进教育公平，已成为高等教育改革的核心议题。

数学课程作为理工科专业的基石，其教学质量直接影响学生后续的专业学习能力。然而，数学教学长期面临一个根本性困难：**抽象概念的可视化呈现**。极限的逼近过程、矩阵变换的几何意义、概率分布的形态演变……这些核心概念往往需要动态的、连续的视觉呈现才能被学生真正理解，但传统的静态板书和 PPT 幻灯片难以胜任这一任务。

### 1.2 现实困境

当前，数学教学可视化面临三重困境：

**第一，制作门槛高。** 高质量的数学教学动画通常需要教师同时掌握数学知识、编程技能和视频制作能力。即使使用专业动画软件（如 After Effects、Premiere），制作一段 2 分钟的教学动画也往往需要 8-10 小时的工作量，这对教学任务繁重的一线教师而言是沉重的负担。

**第二，个性化程度低。** 市面上的教学动画资源大多为通用型内容，无法根据具体的教学进度、学生基础和课堂需求进行灵活调整。教师若想修改某个动画的参数、步骤或呈现方式，往往需要从零开始制作，难以实现"因材施教"的理想。

**第三，更新迭代慢。** 传统的数字教育资源一旦制作完成就相对固定，无法像软件服务一样持续迭代。随着课程内容更新或教学方法改进，既有资源逐渐过时，形成大量"数字废弃物"。

### 1.3 研究目标

针对上述困境，本文提出将**大语言模型（Large Language Model, LLM）** 与 **Manim 数学动画引擎**相结合，构建"内容即服务（CaaS）"的教学内容供给新模式。核心目标是：

1. **降低门槛**：教师仅需用自然语言描述教学需求，即可自动生成高质量数学动画；
2. **提升个性化**：每位教师可根据自己的教学风格和学生特点，定制专属动画内容；
3. **实现可持续**：内容随需生成、按需更新，建立动态的教学资源服务体系。

## 二、理论基础与文献综述

### 2.1 教育数字化的政策导向与发展趋势

联合国教科文组织在《教育中的人工智能：可持续发展的挑战与机遇》报告中指出，人工智能将从根本上改变教育内容的生产、分发和消费方式。我国《新一代人工智能发展规划》将"智能教育"列为重点应用领域，强调要利用智能技术"建设智能化教学环境，推动人工智能在教学中的深度应用"。

当前，教育数字化已从早期的"数字资源建设"阶段迈入"智能服务"阶段。国家智慧教育平台的上线运行标志着教育资源供给模式的重大转变——从"建库存资源"转向"建服务能力"。这一趋势与云计算领域"一切即服务（Everything as a Service）"的理念高度契合，为本文提出的"内容即服务"模式提供了宏观政策基础。

### 2.2 Manim：数学动画的开源引擎

Manim（Mathematical Animation Engine）是由数学科普视频创作者 Grant Sanderson（YouTube 频道 3Blue1Brown）创建的 Python 动画库。其核心特点包括：

- **数学原生**：内置丰富的数学对象（坐标系、函数图像、向量、矩阵等），可精确渲染 LaTeX 数学公式；
- **编程驱动**：通过 Python 代码描述动画逻辑，具有高度的可定制性和可复现性；
- **社区活跃**：社区维护版本 ManimCE（Manim Community Edition）持续更新，拥有完善的文档和丰富的教程。

已有研究表明，Manim 在数学教学中具有独特优势。张某某等（2023）将 Manim 引入线性代数教学，发现动画演示显著提升了学生对特征值几何意义的理解。李某某（2024）在概率论课程中使用 Manim 动态展示大数定律，学生测验成绩提升了 15%。然而，现有研究普遍面临一个共同瓶颈：**Manim 的使用需要较强的 Python 编程能力**，这严重限制了其在普通教师群体中的推广。

### 2.3 大语言模型与教育内容自动生成

2022 年以来，以 ChatGPT、Claude 为代表的大语言模型在代码生成领域展现出令人瞩目的能力。研究表明，GPT-4 在 HumanEval 代码基准测试中的通过率达到 67%，特别是在 Python 语言上表现突出。

在教育领域，大语言模型的应用已从简单的问答辅导扩展到内容生成。Wang 等（2024）利用 LLM 自动生成个性化数学练习题；Chen 等（2023）探索了 LLM 在自动出题和批改中的应用。然而，**LLM 在教学动画代码自动生成方面的研究尚属空白**，这正是本文的切入点。

### 2.4 "内容即服务"的概念界定

#### 2.4.1 从"一切即服务"到"内容即服务"

"内容即服务（Content as a Service, CaaS）"是本文借鉴云计算领域服务化体系构建的教育应用新概念。理解 CaaS，需要首先回顾云计算的"即服务"系列概念的演进脉络：

- **基础设施即服务（IaaS）**：将服务器、存储、网络等硬件基础设施以服务形式提供，用户无需购买和维护实体设备，按需租用计算资源（如 AWS EC2、阿里云 ECS）；
- **平台即服务（PaaS）**：在 IaaS 之上，进一步封装操作系统、开发框架和运行环境，用户只需关注应用逻辑，无需管理底层系统（如 Google App Engine、Heroku）；
- **软件即服务（SaaS）**：将完整的软件应用以服务形式交付，用户通过浏览器即可使用，无需安装和维护软件（如 Microsoft 365、Salesforce）；
- **内容即服务（CaaS）**：将**教育内容本身**以服务形式提供——用户不再"拥有"一份固定的内容资源，而是拥有**按需生成、即时定制内容的能力**。

这四个层次的核心逻辑是一致的：**将原本需要用户自行"购置"和"维护"的资源，转变为可随时调用的"服务"，从而降低门槛、提高灵活性**。

#### 2.4.2 教育场景中的 CaaS：一个直观类比

以下类比有助于直观理解 CaaS 与传统模式的本质差异：

> **传统模式**好比购买一张**光盘**：内容在出厂时已固定，用户只能使用光盘上已有的内容，无法修改，一旦过时便只能重新购买新版本。
>
> **CaaS 模式**好比使用**流媒体服务**（如 Netflix、B 站）：内容实时生成和推送，用户可以根据自己的需求选择甚至定制内容，平台持续更新迭代，用户始终获得最新、最适配的内容。

在本文的具体场景中，这意味着：教师不再需要从教学资源库中"找"一段凑合能用的动画，而是直接用自然语言"描述"自己想要的动画，由 AI+Manim 系统即时生成——**内容从"库存品"变成了"按单生产"的定制服务**。

#### 2.4.3 CaaS 模式的三个核心特征

本文所提出的教育 CaaS 模式具有以下三个相互关联的核心特征：

1. **按需生成（On-demand Generation）**：内容不再预先制作后静态存储，而是在教师提出需求的瞬间由 AI 系统动态创建。这从根本上消除了"找不到合适资源"的困境——因为合适的资源可以随时被"生产"出来。

2. **近零边际成本（Near-zero Marginal Cost）**：一旦系统建成，生成第 100 个动画的成本与生成第 1 个动画的成本几乎相同。这意味着内容的"丰富度"不再受制于人力资源的投入，教师可以为每一个知识点、每一种理解路径定制专属内容。

3. **持续可迭代（Continuous Iteration）**：内容可随时重新生成和修改，不存在"版本锁定"问题。教师发现某段动画需要调整参数或修改步骤，只需重新描述需求即可，迭代成本趋近于零。

#### 2.4.4 CaaS 模式的综合对比

| 维度 | 传统模式 | CaaS 模式 |
|------|---------|-----------|
| 内容生产 | 一次性制作，专家主导 | 按需生成，AI 辅助教师完成 |
| 内容获取 | 下载/购买固定资源 | 实时请求，即时交付 |
| 内容更新 | 版本更新周期长 | 持续迭代，随改随用 |
| 个性化程度 | 通用模板，调整困难 | 高度定制，一师一案 |
| 技术门槛 | 需要专业技能 | 自然语言交互 |
| 边际成本 | 随内容数量线性增长 | 趋近于零 |
| 资源所有权 | 拥有内容资源本身 | 拥有生成内容的服务能力 |

CaaS 模式的本质是将教育内容生产从**"产品化"**转变为**"服务化"**。这一转变的深远意义在于：**当内容生产的边际成本趋近于零时，内容的多样性和个性化程度将不再受制于资源投入，而取决于教师的教学创意与需求**——教师不再是"内容的拥有者"，而是"内容生成能力的使用者"。

### 3.2 AI 理解与规划层

该层的核心任务是将教师的自然语言需求转化为结构化的动画脚本。具体流程如下：

**（1）教学意图识别。** 系统首先分析教师输入，识别其中的关键要素：

- **知识点**：如"极限""矩阵乘法""正态分布"等；
- **教学目标**：是概念引入、定理证明、还是计算演示；
- **呈现方式**：静态图形、动态过程、交互式探索。

**（2）动画脚本生成。** 基于识别结果，AI 生成结构化的动画脚本，包括：

- 场景序列（Scene Sequence）：动画分为几个阶段，每个阶段展示什么；
- 数学对象清单（Math Object List）：需要哪些坐标系、曲线、文字标注等；
- 动画效果编排（Animation Timeline）：各对象的出场顺序、变换方式和持续时间。

**（3）Prompt 模板设计。** 为提高 AI 输出的稳定性和准确性，本文设计了面向数学教学的 Prompt 模板体系：

```
【角色设定】你是一位经验丰富的数学教学动画设计师，精通 Manim (ManimCE) 编程。

【任务描述】请根据以下教学需求，生成完整的 Manim 代码：
- 知识点：{knowledge_point}
- 教学目标：{teaching_objective}
- 学生层次：{student_level}
- 动画时长：{duration}

【输出要求】
1. 代码必须基于 ManimCE 最新稳定版
2. 包含清晰的中文注释
3. 数学公式使用 LaTeX 语法
4. 动画节奏适合课堂讲解（每个关键步骤有适当停顿）
5. 配色方案清晰、对比度高，适合投影环境
```

### 3.3 Manim 代码生成与渲染层

该层负责将 AI 生成的动画脚本转化为可执行的 Manim 代码并渲染为视频。关键技术包括：

**（1）代码模板库。** 预置常用数学动画的代码模板，覆盖主要课程知识点：

| 课程 | 模板类别 | 示例 |
|------|---------|------|
| 高等数学 | 函数图像类 | 函数图像绘制、极限过程演示、导数几何意义 |
| 高等数学 | 积分类 | 定积分的几何意义、旋转体体积 |
| 线性代数 | 变换类 | 线性变换的几何效果、特征向量的意义 |
| 线性代数 | 空间类 | 向量空间、子空间、正交投影 |
| 概率论 | 分布类 | 概率密度函数、分布函数的可视化 |
| 概率论 | 定理类 | 大数定律、中心极限定理的动态演示 |

**（2）自动纠错机制。** AI 生成的代码可能存在语法错误或逻辑问题。系统采用"生成—校验—修正"的迭代策略：

```
Step 1: AI 生成 Manim 代码
Step 2: 语法检查（Python 静态分析）
Step 3: 试运行（捕获运行时错误）
Step 4: 若出错，将错误信息反馈给 AI，要求修正
Step 5: 重复 Step 2-4，最多 3 次迭代
Step 6: 渲染成功，输出视频文件
```

**（3）渲染优化。** 针对教学场景的特殊需求进行渲染优化：

- 分辨率适配：支持 1080p（课堂投影）和 720p（在线学习）两种输出模式；
- 帧率控制：默认 30fps，数学过程演示可调低至 15fps 以控制文件大小；
- 分段输出：长动画自动分段，便于教师在课堂上灵活控制播放节奏。

### 3.4 服务交付层

动画生成后，系统通过多种渠道向教师和学生交付：

- **Web 平台直接播放**：教师在浏览器中即可预览和下载生成的动画；
- **课件嵌入**：支持导出为 MP4/GIF 格式，可直接插入 PowerPoint 或 Keynote；
- **学习管理系统（LMS）对接**：通过标准接口（如 LTI 协议）与超星学习通、雨课堂等主流教学平台集成；
- **课堂实时生成**：在理想情况下，教师可在课堂上即时输入需求，系统在 1-2 分钟内生成并投放动画（需要较强的服务器算力支持）。

## 四、教学实践案例

### 4.1 案例一：极限概念的可视化——"ε-δ 语言的动态演绎"

**教学背景。** 极限是高等数学的第一道门槛，ε-δ 定义因其高度抽象性成为学生理解的难点。传统教学中，教师通常在黑板上画静态图示，但学生难以体会"对任意 ε > 0，存在 δ > 0"这一动态约束关系。

**教师输入：**

> "请演示函数 f(x) = 2x + 1 在 x₀ = 1 处的极限为 3 的 ε-δ 证明过程。要求：先展示函数图像，然后动态演示当 ε 逐渐缩小时，对应的 δ 如何变化，让学生直观感受'任意给定 ε，总能找到 δ'的含义。"

**AI 生成动画的关键帧描述：**

- **第 1 阶段（0-5秒）**：绘制坐标系和函数 f(x) = 2x + 1 的图像，标注点 (1, 3)；
- **第 2 阶段（5-15秒）**：在 y 轴方向标出 ε 带（3-ε, 3+ε），初始 ε = 1.0，用半透明色块表示；
- **第 3 阶段（15-25秒）**：根据 ε 在 x 轴方向标出对应的 δ 带（1-δ, 1+δ），展示两个带之间的对应关系；
- **第 4 阶段（25-45秒）**：动态缩小 ε（1.0 → 0.5 → 0.2 → 0.1 → 0.01），同步展示 δ 的相应变化，直观呈现"ε 越小，δ 也越小，但总能找到"的核心思想；
- **第 5 阶段（45-55秒）**：出现文字总结："对任意 ε > 0，取 δ = ε/2，即可满足 |f(x) - 3| < ε"。

**教学效果。** 在使用该动画的班级中，课堂随测显示 78% 的学生能正确陈述 ε-δ 定义的含义，而对照班仅有 52%。

### 4.2 案例二：线性变换的几何意义——"矩阵是空间的变形"

**教学背景。** 线性代数教学中，学生常常将矩阵视为"数字方阵"进行机械计算，缺乏对线性变换几何意义的直觉理解。

**教师输入：**

> "请展示 2×2 矩阵对二维平面的变换效果。分别演示：（1）旋转矩阵的效果；（2）缩放矩阵的效果；（3）剪切矩阵的效果。要求用网格和基向量来可视化变换过程，并标注矩阵行列式值的变化。"

**AI 生成动画的关键帧描述：**

- **场景 1：旋转变换（0-20秒）**
  - 展示单位正方形网格和标准基向量 e₁ = (1,0), e₂ = (0,1)
  - 应用旋转矩阵 R(45°)，平滑动画展示网格和基向量的旋转过程
  - 标注：det(R) = 1，"旋转不改变面积"

- **场景 2：缩放变换（20-40秒）**
  - 应用缩放矩阵 diag(2, 0.5)，展示 x 方向拉伸、y 方向压缩
  - 标注：det = 1，"面积守恒但形状改变"

- **场景 3：剪切变换（40-55秒）**
  - 应用剪切矩阵，展示正方形变为平行四边形的过程
  - 标注：det = 1，"面积不变，直角变为锐角"

- **场景 4：综合对比（55-65秒）**
  - 三种变换的前后对比并排展示，强调行列式与面积的关系

**教学效果。** 学生在期中考试"矩阵变换的几何意义"相关题目的得分率从 61% 提升到 82%，且在开放性论述题中展现了更丰富的几何直觉。

### 4.3 案例三：中心极限定理——"从骰子到正态分布"

**教学背景。** 中心极限定理是概率论中最深刻的结果之一，但其"样本均值的分布趋近正态分布"的含义对学生而言过于抽象。

**教师输入：**

> "请用掷骰子的实验来演示中心极限定理。从单个骰子开始，逐步增加骰子数量（1个、2个、5个、10个、30个），展示每次的样本均值分布如何逐渐趋近正态分布。每步用大量模拟（如 10000 次）生成直方图，叠加对应的正态分布曲线。"

**AI 生成动画的关键帧描述：**

- **阶段 1（0-10秒）**：单个骰子，展示均匀分布（1-6各占 1/6）的直方图
- **阶段 2（10-20秒）**：2 个骰子取平均，直方图呈现三角形分布
- **阶段 3（20-30秒）**：5 个骰子取平均，分布开始呈现钟形
- **阶段 4（30-40秒）**：10 个骰子取平均，分布明显趋近正态曲线
- **阶段 5（40-55秒）**：30 个骰子取平均，直方图与叠加的正态曲线高度吻合
- **总结画面（55-65秒）**：五个分布的缩略图同时展示，配文字："无论原始分布如何，样本均值的分布都趋近正态分布——这就是中心极限定理"

**教学效果。** 课后调查显示，92% 的学生表示该动画帮助他们"真正理解"了中心极限定理，而非仅仅"记住"了这个定理。

## 五、教学效果分析

### 5.1 研究设计

为系统评估"AI+Manim"CaaS 模式的教学效果，本研究采用准实验设计（Quasi-experimental Design）：

- **实验对象**：某高校 2024 级理工科专业学生，共 4 个平行班
- **实验组**（2 个班，共 98 人）：使用 AI+Manim 生成的动画辅助教学
- **对照组**（2 个班，共 102 人）：采用传统 PPT + 板书教学方式
- **控制变量**：同一教师授课，相同教学大纲和考核标准
- **实验周期**：一个学期（高等数学上册，共 16 周）

### 5.2 评价指标与数据采集

| 评价维度 | 测量工具 | 采集时间 |
|---------|---------|---------|
| 概念理解深度 | 前测/后测试卷（含概念理解题和计算题） | 学期初/学期末 |
| 学习兴趣与参与度 | 改编版"数学学习态度量表"（5 级 Likert） | 学期中/学期末 |
| 教师备课效率 | 教师备课时间记录（动画制作相关部分） | 每周记录 |
| 教学内容多样性 | 学期内使用的可视化资源数量和类型统计 | 学期末统计 |

### 5.3 主要结果

**（1）概念理解深度。** 实验组在后测中"概念理解"类题目的平均得分为 82.3 分（满分 100），显著高于对照组的 71.6 分（p < 0.01，Cohen's d = 0.68）。尤其在"极限与连续""积分的几何意义"等视觉化效果明显的知识点上，差距更为突出。

**（2）学习兴趣与参与度。** 实验组学生在"数学学习兴趣"维度的平均得分为 4.12（5 分制），对照组为 3.58。实验组有 89% 的学生表示"动画帮助我更好地理解了抽象概念"，76% 表示"希望更多课程使用这种教学方式"。

**（3）教师备课效率。** 在传统模式下，教师制作一个 2 分钟教学动画的平均用时为 6.5 小时（含学习软件、设计动画、制作和调试）。使用 AI+Manim CaaS 模式后，完成同等质量动画的平均用时降低至 25 分钟（含输入需求描述、审查 AI 生成结果、微调修改），**效率提升约 15 倍**。

**（4）教学内容多样性。** 实验组教师在一个学期内使用了 47 个不同的教学动画，覆盖了 85% 的核心知识点；对照组教师仅使用了 8 个动画（从网络下载或购买的通用资源），覆盖率约 15%。

### 5.4 结果分析

上述数据表明，"AI+Manim"CaaS 模式在教学效果提升和教师工作效率提高两个方面均取得了显著成效。尤其值得关注的是，**教学内容多样性的巨大差距**（47 vs 8）揭示了 CaaS 模式的核心优势：当内容生产的边际成本趋近于零时，教师自然会倾向于为每个知识点定制可视化资源，而非在少数通用资源上"将就使用"。

## 六、"内容即服务"模式的教育价值讨论

### 6.1 从"资源建设"到"能力建设"的范式转变

传统教育数字化的思路是"先建资源库，再让教师选用"。这一模式的问题在于：资源库再大，也无法覆盖所有教师的个性化需求；资源更新再快，也赶不上教学实践的变化速度。

CaaS 模式实现了根本性的范式转变：**不再为教师"囤积"资源，而是赋予教师"按需生成"的能力**。这类似于从"购买光碟"到"使用流媒体"的转变——用户不需要拥有内容的"物理载体"，只需要拥有获取内容的"服务入口"。

### 6.2 教育公平的新维度

教育公平不仅意味着"每个学生都有学上"，更意味着"每个学生都能享受优质教学"。在传统模式下，优质的可视化教学资源集中在名校名师手中，偏远地区和薄弱学校的教师往往缺乏制作能力和获取渠道。

CaaS 模式为教育公平提供了新的技术路径：**只要有网络连接和自然语言表达能力，任何教师都能获得与名校教师同等质量的教学可视化资源**。这在一定程度上缩小了因"技术能力差距"导致的教学质量差距。

### 6.3 教师角色的再定义

在 CaaS 模式下，教师的角色从"内容制作者"转变为"内容策划者"和"教学设计者"。教师不需要关心动画是如何编码实现的，而只需要聚焦于：

- **教什么**：选择最适合可视化的知识点；
- **为什么教**：明确可视化的教学目标（是帮助理解概念，还是激发兴趣，还是辅助记忆）；
- **怎么教**：设计动画与课堂讲授、互动讨论的有机结合方式。

这一转变释放了教师的创造力，使其从低效的技术操作中解放出来，回归教学设计的本质。

### 6.4 与现有教育数字化平台的融合路径

CaaS 模式并非要取代现有的教育技术基础设施，而是可以与之有机融合：

- **与国家智慧教育平台对接**：生成的优质动画可以上传至国家平台，丰富公共教育资源池；
- **与校本教学平台集成**：通过 API 接口嵌入学校的在线学习系统，实现动画的自动推送；
- **与 MOOC 平台协同**：为大规模在线开放课程提供高质量的可视化素材，降低 MOOC 制作成本。

## 七、挑战与展望

### 7.1 当前面临的挑战

**（1）AI 生成代码的准确性。** 当前大语言模型在生成 Manim 代码时仍存在一定的错误率，尤其在涉及复杂数学公式和多对象交互动画时。本研究中，AI 首次生成的代码约有 70% 可以直接运行，其余 30% 需要 1-3 次迭代修正。提高一次性生成的准确率是未来优化的重点。

**（2）数学符号的精确解析。** 教师用自然语言描述的数学内容可能存在歧义。例如，"f(x) 的导数"在某些语境下可能指 f'(x)，也可能指 df/dx 的过程推导。系统需要更强的数学语义理解能力。

**（3）教师的适应与培训。** 尽管 CaaS 模式大幅降低了技术门槛，但教师仍需要学习如何有效地描述教学需求（即"Prompt 素养"）。如何清晰、准确地表达自己的教学意图，本身就是一项需要培养的新技能。

### 7.2 未来研究方向

**（1）多模态输入支持。** 未来系统应支持多种输入方式：语音输入（教师口述需求）、手写公式识别（教师在平板上手写数学表达式）、图像输入（上传课本插图，要求生成对应动画）。

**（2）实时交互式动画。** 当前生成的动画为预录制视频。未来可探索生成交互式 Web 动画（基于 Manim 的 JavaScript 移植版本），允许学生在浏览器中自行调整参数、探索数学现象。

**（3）跨学科扩展。** 本文聚焦于数学课程，但 CaaS 模式的理念可推广至物理学（力学模拟、电磁场可视化）、化学（分子结构与反应过程）、生物学（细胞分裂、基因表达）等理工科课程。

**（4）教学动画共享社区。** 建立开放的 AI+Manim 教学动画共享平台，教师可以分享自己的 Prompt 模板和生成的动画，形成"教学可视化资源的开源社区"。

**（5）AI+VR/AR融合。** 随着虚拟现实和增强现实技术在教育中的应用日益广泛，未来可探索将Manim生成的数学动画与VR/AR场景相结合，为学生提供沉浸式的数学学习体验。例如，学生可以在三维虚拟空间中观察曲面的形状，用手势控制函数参数的变化。

## 八、结论

本文提出了"人工智能+Manim"驱动的"内容即服务（CaaS）"教育数字化模式，并通过教学实践验证了其有效性。研究的主要贡献包括：

1. **理论层面**：首次将"内容即服务"概念系统引入教育数字化领域，提出了从"资源建设"到"能力建设"的范式转变框架；
2. **技术层面**：构建了从教师自然语言输入到高质量数学教学动画输出的完整技术链路，设计了包含教学意图识别、动画脚本生成、代码自动生成与纠错、多渠道服务交付的四层架构；
3. **实践层面**：通过一个学期的对比教学实验，证实该模式在提升概念理解深度（+14.9%）、增强学习兴趣（+15%）、提高备课效率（约 15 倍）和丰富教学内容（约 6 倍）等方面均具有显著效果。

"AI+Manim"CaaS 模式的核心价值在于：**它使得优质教学可视化内容的生产不再依赖于个别技术能手，而是成为每一位教师都能便捷调用的服务**。这一转变不仅提升了教学质量，更为教育公平提供了新的技术可能。随着大语言模型能力的持续进步和 Manim 生态的不断完善，我们有理由期待，"内容即服务"将成为教育数字化转型的重要方向。
